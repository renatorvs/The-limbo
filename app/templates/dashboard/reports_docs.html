Ingestão por Relatório

Fonte primária de dados — apps em ambientes separados enviam JSON. Sync de banco é secundário.

Como funciona
  1. Agente da app (Sofia, etc.) gera relatório semanal
  2. Envia POST /api/v1/reports/ingest
  3. Limbo extrai KPIs → MVP Cockpit
  4. Resumo vai para RAG → The Oracle entende
  5. Ações propostas → fila HI-C
Prompt para agente da app gerar relatório
Você é o agente de reporting da {{ '{{' }}app_name{{ '}}' }}. Gere um relatório JSON no formato Limbo v1.0 com:
- company_ref.id (UUID cadastrado no Limbo)
- source_app, generated_at, period (última semana)
- executive_summary (3-5 frases)
- domains: cfo, cmo, cpo, cs — cada um com metrics, highlights, risks, recommendations
- actions_proposed (opcional) — ações que precisam aprovação humana

Use dados reais da app. Métricas numéricas em metrics. Narrativa em highlights/risks/recommendations.
